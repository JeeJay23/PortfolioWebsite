Singleton

Voorbeeld van wanneer ik gebruik maak van het singleton pattern. Dit komt uit mijn huidige eigen project. Dit script zit op een gameobject en deze wordt nooit vernietigd. Zo kan ik altijd bij bepaalde waardes.
Link naar de Github. 


using UnityEngine;
using System.Collections;
using System.Collections.Generic;

public class GameManager : MonoBehaviour {

    public static GameManager instance;
    public BoardManager boardScript;
    public bool playersTurn = true;

    private List enemies = new List();
    private int level = 4;

    void Awake()
    {
        if (GameManager.instance == null)
        {
            GameManager.instance = this;
        }
        else
        {
            Destroy(this);
        }

        DontDestroyOnLoad(this);

        boardScript = this.GetComponent();
        InitGame();
    }

    public void GameOver()
    {
        enabled = false;
    }

    public void AddEnemyToList(Enemy enemy)
    {
        enemies.Add(enemy);
    }

    void InitGame()
    {
        boardScript.SetupScene(level);
    }
}

Object-Oriented Programming

Ik ben een card game aan het maken een daarvoor maak ik veel gebruik van OOP. Het idee is dat elke kaart zijn standaardfuncties heeft en dan specifiekere functies gebaseerd op wat de kaart doet. De opstelling van classes ziet er dan zo uit: base class Card -> DamagingCard -> Fireball. Hierbij kan DamagingCard ook bijvoorbeeld HealingCard zijn.

CardCollection namespace, hierin staat alle functionaliteit voor de kaarten. Deck script en het script voor de hand van de speler. De kaarten zelf kunnen nog niks, maar ik kan wel al meerdere kaarten maken en deze toevoegen aan mijn hand.
